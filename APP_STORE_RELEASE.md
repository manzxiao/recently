## 🚀 发布步骤

### Step 1: 构建生产版本

```bash
# 1. 安装/更新 EAS CLI
npm install -g eas-cli

# 2. 登录 Expo 账号
eas login

# 3. 配置 EAS Build（如果首次使用）
eas build:configure

# 4. 构建 iOS 生产版本
eas build --platform ios --profile production
```

**等待时间**：约 10-20 分钟

构建完成后，你会得到：
- 📦 `.ipa` 文件（iOS 安装包）
- 🔗 Expo 构建页面链接

---

### Step 2: 创建 App Store Connect 应用

1. **访问 App Store Connect**
   网址：https://appstoreconnect.apple.com/

2. **创建新 App**
   - 点击"我的 App" → "+"号 → "新建 App"
   - 填写信息：
     ```
     平台：iOS
     名称：时光簿
     主要语言：简体中文
     Bundle ID：com.manzxiao.recently
     SKU：com.manzxiao.recently（或任意唯一标识）
     用户访问权限：完全访问
     ```

3. **填写 App 信息**

### Step 4: 提交审核

#### 4.1 上传构建版本

```bash
# 自动提交到 App Store Connect
eas submit --platform ios --latest
```

或手动操作：
1. 在 App Store Connect 中
2. 进入"时光簿" → "TestFlight" 标签
3. 等待构建版本处理完成（约 5-10 分钟）
4. 构建版本状态变为"可供测试"

#### 4.2 选择构建版本

1. 进入"App Store" 标签
2. 点击左侧"iOS App" 下的版本号（1.0）
3. 在"构建版本"部分，点击"选择构建版本"
4. 选择刚才上传的构建版本

#### 4.3 填写版本信息

**此版本的新增内容**（What's New）：
```
初始版本发布

✨ 核心功能
• 创建和管理倒计时事件
• 支持每日/每周/每月/每年重复提醒
• 桌面小组件实时显示
• 数据备份和恢复
• 日式极简设计

感谢您的使用！
```

#### 4.4 内容版权信息

- **广告标识符（IDFA）**：否
- **内容版权**：根据实际情况选择（个人 App 选"否"）

#### 4.5 App 审核信息

| 字段 | 建议内容 |
|------|---------|
| **联系信息** | 你的邮箱和电话 |
| **演示账号** | 不需要（App 无登录） |
| **备注** | （可选）留空或说明 App 的主要功能 |

#### 4.6 版本发布方式

选择以下之一：
- **自动发布**：审核通过后立即上线
- **手动发布**：审核通过后等你手动点击发布
- **定时发布**：审核通过后在指定时间发布

**建议**：首次发布选择"手动发布"，以便审核通过后你可以再次检查。

#### 4.7 提交审核

1. 点击右上角"存储"
2. 点击"提交以供审核"
3. 回答导出合规性问题：
   - "您的 App 是否使用加密？" → **否**
   - （因为 app.json 中设置了 `ITSAppUsesNonExemptEncryption: false`）
4. 确认提交

---

### Step 5: 等待审核

#### 审核时间线

- **处理中**：1-2 小时（Apple 开始审核）
- **正在审核**：1-3 天（实际审核时间）
- **等待开发者发布** 或 **准备销售**：审核通过

#### 常见被拒原因

1. **截图与实际功能不符**
   - 解决：确保截图是真实 App 界面

2. **隐私政策链接无效**
   - 解决：确保 PRIVACY.md 已上传到可访问的 URL

3. **崩溃或功能异常**
   - 解决：提交前充分测试

4. **元数据问题**
   - 解决：检查 App 名称、描述是否有误导性内容
